📢 自定义Shared Element Transition注意事项

Activity Shared Element Transition (ASET) 和Fragment Shared Element Transition (FSET) 是有本质不同的。因此，自定义Transition想要同时兼容这两种模式，其实并不简单。

内容可能有错误或瑕疵，待完善…

从A到B的ASET

系统会将B中View的一些属性设为A中View的值，然后根据初始值和终值的不同构造执行Transition。如果你自定义的Transition旨在Activity间使用，那么在captureStartValues中获取初始值时需要注意，此时获取到的View是Activity B中共享元素，而不是Activity A中的那个共享元素。能获取到的信息只有位置、大小、可见性等信息，其他信息无法获得，只能通过xml或代码指定。但是在SharedElmentCallback#onSharedElementStart()，可以获取到Activity A的元素的Snapshot Bitmap ，此View是源共享元素的快照，可以将其添加到Overlay中伪造源View。

为什么要这样设计？

原因在于Activity之间的Transition数据是通过Intent传递的，所以不可能将整个View传递到下一个Activity，只能传递某些可以序列化的属性，如大小、位置、快照等。虽然有局限性，但是也有其优越性：这意味着，只要对接完好，即使是两个不同应用的Activity也可以进行共享元素Transition。

Shared Element Return Transition

Shared Element Return Transition (SERT) 在ASET中也是非常特殊的存在，需要注意以下问题。

初始状态问题

对于SERT，需要注意的是，在SERT之前系统会把视图设置为SERT的end状态（也是Shared Elment Enter Transition的start状态）保证captureEndValues能正常工作。因此，如果要执行动画，必须将视图手动重置为SERT的start状态。比如，你需要执行一个位移动画，那么，请先通过view.layout将View移动到初始位置，动画才能如期（as expected）执行。

最后一帧问题

需要知道的是，Transition都是在 Activity B中执行的。在SERT执行完成之后，B会被finish。考虑到这一点，需要特别注意动画的onAnimationEnd回调是否有改变共享元素外观的代码（比如移除Overlay中的Drawable/View），因为onAnimationEnd是在动画最后一帧执行的，在onAnimationEnd和finish之间就会有一帧的空隙，如果不处理好，会出现视图一闪而过的突变，体验非常糟糕。

从A到B的FSET

但是对于FSET，在captureValues中获取到的View就分别是A中的共享元素和B中共享元素。通过captureValues获到任何需要的初始值和终值。当然，对于从View->View的Transition更是如此，因此，构造动画基本上不需要外部指定数据。

不同于ASET: Enter Transition在B中执行，Return Transition在A中执行，FSET的Transition都是在目标Fragment中执行，对视图执行的动画最后都会停留在布局中，而不会像ASET在执行完后因为finish而销毁。所以需要考虑的问题就没有那么多。
